# Codalon — Proposal Pill Spec v1.0

**Status:** Closed
**Validated by:** Oli
**Date:** 2026-03-16

---

## 1. Component Identity

**Component name:** `CodalonProposalPill`
**Type:** Transient overlay, renders on `Proposal Layer` (z-index below HUD, above Canvas)
**Scope:** Root shell only. One pill visible at a time. Never shown during onboarding.

---

## 2. Trigger Conditions

The pill appears when **all** of the following are true:

| Condition | Detail |
|---|---|
| Rule engine emits a context proposal | Via `EventBus` signal — event type `ContextProposalEvent` |
| Proposed context differs from active context | No pill if already in the proposed context |
| No pill currently visible | New proposal replaces existing pill only if confidence delta is significant — threshold defined in Epic 17 |
| Developer is not mid-interaction | No pill during: active sheet, active popover, context transition in progress |

|---|---|

| Proposed context | Border tint |

## 5. Content Layout

```
[mode icon]  [signal description]  ·  [Switch]  [✕]
```

Full-width `HStack(spacing: 8)`, 12 pt horizontal padding.

| Element | Spec |
|---|---|
| Mode icon | SF Symbol, 12×12 pt, proposed context colour |
| Signal description | `Text`, 12pt, `.regular` weight, `primary.opacity(0.85)`, single line, truncates with `...` |
| Dot separator | `Text("·")`, 12pt, `secondary` colour |
| Switch button | `Text("Switch")`, 12pt, `.medium` weight, proposed context colour |
| Dismiss button | SF Symbol `xmark`, 10×10 pt, `secondary` colour, tap target min 28×28 pt |

**Example rendered content:**
```
⬡  Release Mode suggested · active build detected  ·  Switch  ✕
```

Signal description text is generated by the rule engine alongside the proposal event. The pill renders whatever string it receives — it owns no detection logic.

## 6. Appearance Animation

| Property | Value |
|---|---|
| Entry | Slides down from title bar edge: `translateY(-48 → 0)` + `opacity(0 → 1)` |
| Duration | 280ms |
| Curve | `.spring(response: 0.35, dampingFraction: 0.78)` — slight overshoot |
| Entry delay | 0.2s after trigger — prevents flash on rapid signal changes |

---

## 7. Dismissal Behaviour

### 7.1 Auto-dismiss

| Property | Value |
|---|---|
| Timeout | 8.0s from appearance |
| Timer resets | Never — timer is not reset by hover or interaction elsewhere |
| Animation | `opacity(1 → 0)` + `translateY(0 → -12)`, 200ms, `.easeIn` |

### 7.2 Manual Dismiss (✕)

| Action | Result |
|---|---|
| Tap ✕ | Pill dismisses with same animation as auto-dismiss |
| Post-dismiss | Proposal suppressed until rule engine emits new high-confidence signal (Epic 17 threshold) |

### 7.3 Switch Action

| Action | Result |
|---|---|
| Tap "Switch" | Pill dismisses immediately (100ms fade) → context transition fires (Shell Spec §7.2) |
| Post-switch | Manual selection persists, auto-detection proposals suppressed until significant signal change |

### 7.4 Ignored (auto-dismissed, no interaction)

| Post-dismiss | Proposal suppressed for cooldown period — duration defined in Epic 17 |
|---|---|

### 7.5 Replace Behaviour (multiple rapid proposals)

**Validated decision:** Incoming proposal immediately replaces outgoing pill — no queue. Outgoing fades out, incoming slides in. Replace only triggers if new proposal has meaningfully higher confidence than dismissed one. Threshold defined in Epic 17.

---

## 8. HUD Interaction

When the pill is visible:

- HUD strip is **force-visible** — auto-hide timer suspended
- Proposal ring on HUD Center Zone **activates** simultaneously with pill appearance
- When pill dismisses (any method): HUD returns to normal auto-hide, proposal ring deactivates

Both elements are driven by the same `ContextProposalState` in the environment and are always in sync.

---

## 9. State Matrix

| Pill state | HUD ring | HUD force-visible | Auto-hide timer |
|---|---|---|---|
| Not shown | Off | No | Normal |
| Visible, counting down | On | Yes | Running (8s) |
| Dismissed (✕ or timeout) | Off | No | Resumes |
| Accepted (Switch) | Off | No | Resumes |
| Blocked (sheet/popover open) | Off | — | Not started |

## 11. SwiftUI Component Sketch

Structure reference only — not final implementation.

```swift
// CodalonProposalPill.swift

struct CodalonProposalPill: View {
    let proposal: ContextProposal
    let onSwitch: () -> Void
    let onDismiss: () -> Void

    @State private var isVisible: Bool = false
    @State private var dismissTask: Task<Void, Never>?

    var body: some View {
        HStack(spacing: 8) {
            Image(systemName: proposal.context.iconName)
                .font(.system(size: 12))
                .foregroundStyle(proposal.context.tintColor)

            Text(proposal.signalDescription)
                .font(.system(size: 12))
                .lineLimit(1)
                .foregroundStyle(.primary.opacity(0.85))

            Text("·")
                .foregroundStyle(.secondary)

            Button("Switch", action: onSwitch)
                .font(.system(size: 12, weight: .medium))
                .foregroundStyle(proposal.context.tintColor)

            Button(action: onDismiss) {
                Image(systemName: "xmark")
                    .font(.system(size: 10))
                    .foregroundStyle(.secondary)
            }
            .frame(minWidth: 28, minHeight: 28)
        }
        .padding(.horizontal, 12)
        .frame(width: 420, height: 36)
        .background(.regularMaterial)
        .clipShape(Capsule())
        .overlay(
            Capsule()
                .strokeBorder(proposal.context.tintColor.opacity(0.6), lineWidth: 0.5)
        )
        .shadow(color: .black.opacity(0.10), radius: 8, x: 0, y: 4)
        .opacity(isVisible ? 1 : 0)
        .offset(y: isVisible ? 0 : -48)
        .animation(
            .spring(response: 0.35, dampingFraction: 0.78),
            value: isVisible
        )
        .onAppear { startSequence() }
        .buttonStyle(.plain)
    }

    private func startSequence() {
        Task {
            try? await Task.sleep(for: .seconds(0.2))
            isVisible = true
        }
        dismissTask = Task {
            try? await Task.sleep(for: .seconds(8))
            onDismiss()
        }
    }
}
```

`ContextProposal` is a value type owned by the rule engine (Epic 17). The pill owns no business logic — renders and reports actions only.
